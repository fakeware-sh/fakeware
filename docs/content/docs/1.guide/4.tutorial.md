This tutorial builds a realistic shop dataset from scratch: a tax rate, a category tree, a hundred products, customers, and orders that tie them together. Along the way you'll use every core feature: `define`, `many`, `ref`, `shop`, seeded randomness, and the order builders.

We'll use [Faker](https://fakerjs.dev) to generate believable values. Install it into your project:

::code-group
```bash [npm]
npm install -D @faker-js/faker
```

```bash [pnpm]
pnpm add -D @faker-js/faker
```

```bash [yarn]
yarn add -D @faker-js/faker
```

```bash [bun]
bun add -d @faker-js/faker
```

```bash [deno]
deno add -D npm:@faker-js/faker
```
::

Each step adds one file to `data/`. Fakeware evaluates every file, so you can split your dataset however you like. The order of files doesn't matter, because references drive the write order.

## 1. Tax

Start with a tax rate. Every product needs one, so this is the natural root of the dependency graph.

```ts [data/tax.ts]
import { define } from '@fakeware/core'

define('tax', {
  $key: 'standard',
  name: 'Standard rate',
  taxRate: 19,
})
```

The `$key` is a stable, human-readable handle. Other entities reference this record with `ref('tax').key('standard')`, so you never deal with raw UUIDs.

## 2. Categories

Next, a small category tree. Categories are **self-referential**: a child points at its parent. Fakeware sorts records *within* an entity so parents are written before children.

```ts [data/categories.ts]
import { define, ref } from '@fakeware/core'

define('category', [
  { $key: 'root', name: 'Catalog' },
  { $key: 'bikes', name: 'Bikes', parentId: ref('category').key('root') },
  { $key: 'apparel', name: 'Apparel', parentId: ref('category').key('root') },
  { $key: 'accessories', name: 'Accessories', parentId: ref('category').key('root') },
])
```

::note
`ref('category').key('root')` inside the `category` entity is a self-reference. Fakeware detects the parent-before-child ordering automatically, so no manual sorting is needed.
::

## 3. Products

Now the catalog. We'll generate a hundred products with `many`, give each a Faker-generated name and price, tie them to the standard tax, and assign each a random handful of categories.

```ts [data/products.ts]
import { faker } from '@faker-js/faker'
import { define, many, ref, shop } from '@fakeware/core'

const PRODUCT_COUNT = 100

define(
  'product',
  many(PRODUCT_COUNT, (ctx) => {
    const gross = faker.number.float({ min: 5, max: 1500, fractionDigits: 2 })

    return {
      name: faker.commerce.productName(),
      productNumber: `SW-${100000 + ctx.index}`,
      stock: faker.number.int({ min: 0, max: 500 }),
      active: true,
      description: faker.commerce.productDescription(),
      taxId: ref('tax').key('standard'),
      price: [{ currencyId: shop.defaultCurrency, gross, net: gross / 1.19, linked: true }],
      categories: ref('category').pick(faker.number.int({ min: 1, max: 3 })),
    }
  }),
)
```

Three references at work here:

::field-group
  :::field{name="ref('tax').key('standard')"}
  A one-to-one reference by key: every product uses the same tax.
  :::

  :::field{name="shop.defaultCurrency"}
  A shop lookup that resolves to the store's default currency at apply time.
  :::

  :::field{name="ref('category').pick(n)"}
  A seeded random pick of `n` categories, returned as an association list. The same product always picks the same categories.
  :::
::

::tip
`ctx.index` is the zero-based position within the `many` batch. Use it for stable, unique fields like product numbers so re-runs don't churn.
::

## 4. Customers

Orders need customers to belong to. Generate a batch with Faker:

```ts [data/customers.ts]
import { faker } from '@faker-js/faker'
import { define, many, shop } from '@fakeware/core'

define(
  'customer',
  many(50, (ctx) => {
    const firstName = faker.person.firstName()
    const lastName = faker.person.lastName()

    return {
      customerNumber: `C-${1000 + ctx.index}`,
      email: `customer${ctx.index}@example.com`,
      firstName,
      lastName,
      salutationId: shop.salutation('mr'),
      defaultPaymentMethodId: shop.defaultPaymentMethod,
      salesChannelId: shop.defaultSalesChannel,
      languageId: shop.defaultLanguage,
    }
  }),
)
```

Here `shop.salutation('mr')` looks a salutation up by its key, while the `default*` tokens grab the shop's defaults. If a lookup fails, Fakeware tells you exactly which keys *are* available.

## 5. Orders

Orders are the most structured entity in Shopware: they bundle line items, addresses, deliveries and payments, each with computed prices. Fakeware ships **order builders** so you don't assemble that by hand.

`builders(ctx)` gives you a set of helpers that generate correctly-shaped, correctly-priced sub-records and stable local IDs:

```ts [data/orders.ts]
import { faker } from '@faker-js/faker'
import { builders, define, many, ref, shop } from '@fakeware/core'

const ORDER_COUNT = 100
const PRODUCT_COUNT = 100

define(
  'order',
  many(ORDER_COUNT, (ctx) => {
    const b = builders(ctx)
    const firstName = faker.person.firstName()
    const lastName = faker.person.lastName()

    const lineItems = b.lineItems.products(
      Array.from({ length: faker.number.int({ min: 1, max: 5 }) }, () => {
        const productIndex = faker.number.int({ min: 0, max: PRODUCT_COUNT - 1 })
        return {
          product: ref('product').at(productIndex),
          label: faker.commerce.productName(),
          unitPrice: faker.number.float({ min: 5, max: 800, fractionDigits: 2 }),
          quantity: faker.number.int({ min: 1, max: 4 }),
          productNumber: `SW-${100000 + productIndex}`,
        }
      }),
    )

    const shippingCost = faker.helpers.arrayElement([4.99, 6.99, 0])

    const billing = b.address({
      firstName,
      lastName,
      salutationId: shop.salutation('mr'),
      street: faker.location.streetAddress(),
      zipcode: faker.location.zipCode(),
      city: faker.location.city(),
      countryId: shop.country('DE'),
    })

    return b.order({
      orderNumber: `${10000 + ctx.index}`,
      salesChannelId: shop.defaultSalesChannel,
      languageId: shop.defaultLanguage,
      currencyId: shop.currency('EUR'),
      orderCustomer: {
        customerId: ref('customer').pick(),
        email: `customer${ctx.index}@example.com`,
        firstName,
        lastName,
        salutationId: shop.salutation('mr'),
      },
      billing,
      lineItems,
      shippingCost,
      deliveries: [
        b.delivery({
          ship: billing,
          method: shop.shippingMethod('standard'),
          cost: shippingCost,
        }),
      ],
      payment: b.payment({
        method: shop.defaultPaymentMethod,
        amount: lineItems.reduce((sum, li) => sum + li.price.totalPrice, 0) + shippingCost,
      }),
    })
  }),
)
```

What the builders do for you:

::field-group
  :::field{name="b.lineItems.products(items)"}
  Turns a compact list of products into fully-formed order line items with calculated prices and price definitions.
  :::

  :::field{name="b.address(input)"}
  Builds an address record with a stable local ID you can reuse for billing and shipping.
  :::

  :::field{name="b.delivery(input)"}
  Builds a delivery with the shipping method, address, and calculated shipping costs. Defaults the delivery state to `open`.
  :::

  :::field{name="b.payment(input)"}
  Builds a transaction with the payment method and calculated amount. Defaults the transaction state to `open`.
  :::

  :::field{name="b.order(input)"}
  Assembles the order: sets the order state, computes the cart price from the line items and shipping, and wires up addresses, deliveries and transactions.
  :::
::

::note
`ref('product').at(productIndex)` references a product by its position in the batch, and `ref('customer').pick()` picks one customer at random (seeded, so it's stable). See [References](/docs/concepts/references) for every reference form.
::

## Apply the whole dataset

Your `data/` directory now holds five files. Preview, then apply:

```bash [Terminal]
fakeware up --dry-run
fakeware up
```

Fakeware resolves everything, orders the writes (tax, categories, products, customers, then orders), and syncs them in one pass per entity. Open your admin and browse the catalog and orders.

## Iterate safely

Change a value (say, bump `PRODUCT_COUNT` or tweak a price) and run `up` again. Only the records that actually changed are rewritten; everything else is left untouched. When you want a clean slate:

```bash [Terminal]
fakeware down
```

Every record Fakeware created, across all five files, is removed, and nothing else.
