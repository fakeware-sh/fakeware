Orders are the most structured entity in Shopware: an order bundles line items, addresses, deliveries and transactions, each with computed prices and states. The order builders assemble all of that correctly so you don't hand-write it.

```ts
import { builders } from '@fakeware/core'
```

## `builders(ctx)`

Call `builders(ctx)` inside a generator to get a set of helpers bound to that record. They share a local ID allocator, so the sub-records they produce get stable, unique IDs that reference each other correctly.

```ts
define(
  'order',
  many(100, (ctx) => {
    const b = builders(ctx)
    // b.address, b.lineItems, b.delivery, b.payment, b.order
  }),
)
```

The default tax rate for all price calculations is **19%**; every builder that computes a price accepts an optional tax override as its last argument.

## `b.address(input)`

Build an address record with a stable local ID. Reuse the returned record for both billing and shipping.

::field-group
  :::field{name="firstName, lastName" type="string"}
  The recipient's name.
  :::

  :::field{name="street, zipcode, city" type="string"}
  The postal address.
  :::

  :::field{name="salutationId" type="Id"}
  Typically `shop.salutation('mr')`.
  :::

  :::field{name="countryId" type="Id"}
  Typically `shop.country('DE')`.
  :::
::

```ts
const billing = b.address({
  firstName: 'Ada',
  lastName: 'Lovelace',
  salutationId: shop.salutation('mr'),
  street: 'Analytical Way 1',
  zipcode: '10115',
  city: 'Berlin',
  countryId: shop.country('DE'),
})
```

Returns an `AddressRecord` (the input plus an `id`).

## `b.lineItems.products(items, tax?)`

Turn a compact product list into fully-formed order line items with calculated prices.

Each item:

::field-group
  :::field{name="product" type="Id"}
  Reference to the product, e.g. `ref('product').at(i)`.
  :::

  :::field{name="label" type="string"}
  Display label for the line.
  :::

  :::field{name="unitPrice" type="number"}
  Gross unit price.
  :::

  :::field{name="quantity" type="number"}
  Defaults to `1`.
  :::

  :::field{name="productNumber" type="string"}
  Optional; added to the line payload.
  :::
::

```ts
const lineItems = b.lineItems.products([
  { product: ref('product').at(0), label: 'Trailblazer', unitPrice: 1299, quantity: 1 },
  { product: ref('product').at(4), label: 'Helmet', unitPrice: 79.9, quantity: 2 },
])
```

Each returned `LineItemRecord` carries a calculated `price` (with `totalPrice`), a `priceDefinition`, and position: everything Shopware expects for an order line item.

## `b.delivery(input, tax?)`

Build a delivery with its shipping method, address and calculated shipping cost.

::field-group
  :::field{name="ship" type="AddressRecord"}
  The shipping address; pass the `b.address(...)` you built.
  :::

  :::field{name="method" type="Id"}
  Shipping method, e.g. `shop.shippingMethod('standard')`.
  :::

  :::field{name="cost" type="number"}
  Shipping cost. Defaults to `0`.
  :::

  :::field{name="earliest, latest" type="string"}
  Optional ISO date strings for the shipping window.
  :::

  :::field{name="stateId" type="Id"}
  Optional. Defaults to the delivery state `open`.
  :::
::

```ts
b.delivery({
  ship: billing,
  method: shop.shippingMethod('standard'),
  cost: 4.99,
})
```

## `b.payment(input, tax?)`

Build a transaction (payment) with its method and calculated amount.

::field-group
  :::field{name="method" type="Id"}
  Payment method, e.g. `shop.defaultPaymentMethod`.
  :::

  :::field{name="amount" type="number"}
  Gross amount paid.
  :::

  :::field{name="stateId" type="Id"}
  Optional. Defaults to the transaction state `open`.
  :::
::

```ts
b.payment({
  method: shop.defaultPaymentMethod,
  // sum of line-item totals + shipping
  amount: lineItems.reduce((sum, li) => sum + li.price.totalPrice, 0) + shippingCost,
})
```

## `b.order(input)`

Assemble the order itself. It computes the cart price from the line items and shipping, sets the order state (default `open`), adds the rounding config Shopware expects, and wires up addresses, deliveries and transactions.

::field-group
  :::field{name="orderNumber" type="string"}
  The human-readable order number.
  :::

  :::field{name="billing" type="AddressRecord"}
  The billing address; also added to the order's address list.
  :::

  :::field{name="lineItems" type="LineItemRecord[]"}
  From `b.lineItems.products(...)`.
  :::

  :::field{name="deliveries" type="DeliveryRecord[]"}
  Optional; from `b.delivery(...)`.
  :::

  :::field{name="payment" type="TransactionRecord"}
  Optional; from `b.payment(...)`.
  :::

  :::field{name="shippingCost" type="number"}
  Optional; folded into the cart price.
  :::

  :::field{name="tax" type="number"}
  Optional tax rate for price calculation. Defaults to `19`.
  :::
::

Any other fields you pass (`salesChannelId`, `languageId`, `currencyId`, `orderCustomer`, `orderDateTime`, …) are carried straight through onto the order.

```ts
return b.order({
  orderNumber: `${10000 + ctx.index}`,
  salesChannelId: shop.defaultSalesChannel,
  languageId: shop.defaultLanguage,
  currencyId: shop.currency('EUR'),
  orderCustomer: {
    customerId: ref('customer').pick(),
    email: 'ada@example.com',
    firstName: 'Ada',
    lastName: 'Lovelace',
    salutationId: shop.salutation('mr'),
  },
  billing,
  lineItems,
  shippingCost: 4.99,
  deliveries: [b.delivery({ ship: billing, method: shop.shippingMethod('standard'), cost: 4.99 })],
  payment: b.payment({
    method: shop.defaultPaymentMethod,
    amount: lineItems.reduce((sum, li) => sum + li.price.totalPrice, 0) + 4.99,
  }),
})
```

## `assocIds`

The lower-level helper the builders use to allocate stable local IDs from the record seed. You rarely need it directly, but it's exported for building your own composite records:

```ts
import { assocIds } from '@fakeware/core'

const ids = assocIds(ctx)
const id = ids.next('myAddress')
```

Calling `next(name)` yields a deterministic ID for that name within the record, so composite structures stay stable across runs.

## A note on prices

The builders compute prices with the [price helpers](/docs/usage/reference/price-helpers). If you need to build order-adjacent structures by hand, those helpers (`price.calculated`, `price.cart`, `price.gross`) produce the exact shapes Shopware expects. They're imported from `@fakeware/core/shopware`.
