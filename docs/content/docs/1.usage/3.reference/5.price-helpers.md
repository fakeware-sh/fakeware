Shopware has several different price shapes depending on where a price appears: a product price, an order line item's calculated price, and an order's cart price each look different. The `price` helper produces each shape correctly, with the tax math done for you.

```ts
import { price } from '@fakeware/core/shopware'
```

All helpers round money to two decimals, and net values are derived from gross using the tax rate you provide.

## `price.gross(unit, options)`

A single-currency **gross product price**: the shape you put in a product's `price` array.

::field-group
  :::field{name="unit" type="number"}
  The gross unit price.
  :::

  :::field{name="options.tax" type="number"}
  The tax rate, used to compute net.
  :::

  :::field{name="options.currencyId" type="string | ShopToken"}
  Optional. Defaults to `shop.defaultCurrency`.
  :::
::

Returns `{ currencyId, gross, net, linked: true }`.

```ts
define('product', {
  name: 'Trailblazer',
  price: [price.gross(1299, { tax: 19 })],
})
```

## `price.calculated(unit, taxRate, options)`

A **calculated price**: the shape used by order line items, deliveries and transactions. It includes the computed tax breakdown Shopware expects.

::field-group
  :::field{name="unit" type="number"}
  The gross unit price.
  :::

  :::field{name="taxRate" type="number"}
  The tax rate.
  :::

  :::field{name="options.qty" type="number"}
  Optional quantity. Defaults to `1`; the total is `unit x qty`.
  :::
::

Returns `{ unitPrice, quantity, totalPrice, calculatedTaxes, taxRules, referencePrice: null, listPrice: null }`.

```ts
const linePrice = price.calculated(79.9, 19, { qty: 2 })
```

::note
The order builders call `price.calculated` for you on line items, deliveries and payments. You only need it directly when assembling order structures by hand.
::

## `price.cart(positions, shipping, taxRate)`

An order's **cart price**: the top-level `price` on an order, combining line item positions and shipping.

::field-group
  :::field{name="positions" type="number"}
  Sum of the line item totals.
  :::

  :::field{name="shipping" type="number"}
  Shipping cost to add.
  :::

  :::field{name="taxRate" type="number"}
  The tax rate.
  :::
::

Returns `{ netPrice, totalPrice, rawTotal, positionPrice, taxStatus: 'gross', calculatedTaxes, taxRules }`.

```ts
const positions = lineItems.reduce((sum, li) => sum + li.price.totalPrice, 0)
const cart = price.cart(positions, 4.99, 19)
```

::note
`b.order(...)` computes the cart price for you. Use `price.cart` directly only if you're building an order without the [order builders](/docs/usage/reference/order-builders).
::

## `price.net(gross, taxRate)`

The low-level helper the others build on: compute a **net** value from a gross value and tax rate.

```ts
price.net(119, 19) // 100
```

Useful when you're filling in a `net` field yourself:

```ts
const gross = 19.99
price: [{ currencyId: shop.defaultCurrency, gross, net: price.net(gross, 19), linked: true }]
```

## Choosing the right helper

::field-group
  :::field{name="Product price"}
  `price.gross(unit, { tax })` goes in the product's `price` array.
  :::

  :::field{name="Order line / delivery / payment"}
  `price.calculated(unit, tax, { qty })`, or let the order builders do it.
  :::

  :::field{name="Order total"}
  `price.cart(positions, shipping, tax)`, or let `b.order` do it.
  :::

  :::field{name="Just the net number"}
  `price.net(gross, tax)`.
  :::
::
