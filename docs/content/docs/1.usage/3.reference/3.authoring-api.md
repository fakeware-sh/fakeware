Everything you import from `@fakeware/core` to author data. These are the functions you use in `data/*.ts` files.

```ts
import { define, many, keyed, ref, shop } from '@fakeware/core'
```

::note
Every authoring function above is exported from the package root, `@fakeware/core`. `defineConfig` lives on `@fakeware/core/config`, and the Shopware helpers (`price`, `createShopwareClient`, …) live on `@fakeware/core/shopware`. `shop` is exported from both the root and `/shopware`. See [Shopware API](/docs/usage/reference/shopware-api) for the subpath details.
::

## `define`

```ts
define(entity, records)
```

Register one or more records for a Shopware entity.

::field-group
  :::field{name="entity" type="string"}
  The Shopware entity's technical name (`product`, `category`, `order`, …).
  :::

  :::field{name="records" type="Record | Record[]"}
  A single record or an array. Each record is a plain object of Shopware fields, plus the optional `$key`, and may itself be a function of `ctx`.
  :::
::

Returns a **key map**: an object whose properties are the `$key`s you defined, each mapped to a reference token. This is mostly used internally; you reference records by `ref('entity').key('…')` rather than through the return value.

```ts
define('tax', { $key: 'standard', name: 'Standard', taxRate: 19 })

define('category', [
  { $key: 'root', name: 'Catalog' },
  { $key: 'bikes', name: 'Bikes', parentId: ref('category').key('root') },
])
```

::note
`$key` is stripped before writing to Shopware. It seeds the record's deterministic ID and lets other records reference it. Keys must be unique within an entity.
::

## `many`

```ts
many(n, fn)
```

Produce `n` generator functions from a single factory. Pass the result as the `records` argument to `define`.

::field-group
  :::field{name="n" type="number"}
  How many records to produce.
  :::

  :::field{name="fn" type="(ctx: Ctx) =&gt; Record"}
  Called once per record, with the generator context.
  :::
::

```ts
define(
  'product',
  many(100, (ctx) => ({
    name: `Product ${ctx.index + 1}`,
    productNumber: `SW-${10000 + ctx.index}`,
  })),
)
```

## `keyed`

```ts
keyed(list, keyFn)
```

Attach a `$key` to each item of a list, so every item becomes referenceable. Duplicate keys are automatically de-duplicated with a numeric suffix.

::field-group
  :::field{name="list" type="readonly T[]"}
  The items to key.
  :::

  :::field{name="keyFn" type="(item: T, index: number) =&gt; string"}
  Produces the base key for each item.
  :::
::

```ts
define(
  'property_group_option',
  keyed(
    ['Red', 'Green', 'Blue'].map((name) => ({ name })),
    (item) => item.name.toLowerCase(),
  ),
)
```

## `ref`

```ts
ref(entity)
```

Return a builder for referencing records of `entity`. The builder has four methods:

::field-group
  :::field{name=".key(key)" type="=&gt; RefToken"}
  A single record by its `$key`.
  :::

  :::field{name=".at(index)" type="=&gt; RefIndexToken"}
  A single record by its position in the batch.
  :::

  :::field{name=".pick(count?)" type="=&gt; PickToken"}
  A seeded random selection: one record with no argument, or an association list of `count` distinct records.
  :::

  :::field{name=".all()" type="=&gt; RefsToken"}
  Every record of the entity, as an association list.
  :::
::

```ts
taxId: ref('tax').key('standard')
product: ref('product').at(0)
manufacturerId: ref('product_manufacturer').pick()
categories: ref('category').pick(3)
allCategories: ref('category').all()
```

Resolution errors (missing key, out-of-range index, empty pick) are reported with the entity, what you asked for, and what's available. See [References](/docs/usage/concepts/references) for the full model.

## The generator context: `Ctx`

Passed to every generator function (records used as functions, and `many` factories):

::field-group
  :::field{name="index" type="number"}
  Zero-based position of this record within its batch.
  :::

  :::field{name="count" type="number"}
  Total number of records in the batch.
  :::

  :::field{name="seed" type="number"}
  Deterministic seed derived from the entity and index. Drives seeded picks; use it for your own stable randomness.
  :::

  :::field{name="shop" type="Shop"}
  The live shop context (same data as the `shop` object). Available because generators run after context is fetched.
  :::
::

```ts
many(50, (ctx) => ({
  productNumber: `SW-${10000 + ctx.index}`,
  isLast: ctx.index === ctx.count - 1,
}))
```

## Token types

The reference builders return **tokens**: small tagged objects Fakeware resolves during the build-plan stage. You rarely handle these directly, but they appear in type signatures:

::field-group
  :::field{name="RefToken"}
  Produced by `.key()`: one record by key.
  :::

  :::field{name="RefIndexToken"}
  Produced by `.at()`: one record by index.
  :::

  :::field{name="PickToken"}
  Produced by `.pick()`: a seeded selection.
  :::

  :::field{name="RefsToken"}
  Produced by `.all()`: all records.
  :::

  :::field{name="ShopToken"}
  Produced by any `shop.*` lookup: resolved against the live shop.
  :::
::

For building orders specifically, the [order builders](/docs/usage/reference/order-builders) generate correctly-shaped line items, addresses, deliveries and payments so you don't assemble them by hand.
