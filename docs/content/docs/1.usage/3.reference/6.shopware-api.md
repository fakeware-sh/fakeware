The `@fakeware/core/shopware` subpath exposes the lower-level building blocks Fakeware uses to talk to Shopware. Most projects never import from it directly (the CLI wires it up), but it's here when you need the client, the price helpers, or the shop lookups outside a data file.

```ts
import {
  price,
  shop,
  createShopwareClient,
  createSyncSink,
  validateConnection,
  withRetry,
} from '@fakeware/core/shopware'
```

## `shop`

The shop lookup object. Every member returns a **token** that is resolved against the live shop during a run. Also re-exported from `@fakeware/core` for use in data files.

### Defaults

```ts
shop.defaultCurrency
shop.defaultLanguage
shop.defaultSalesChannel
shop.defaultCountry
shop.defaultSalutation
shop.defaultTax
shop.defaultPaymentMethod
shop.defaultShippingMethod
```

### Lookups

::field-group
  :::field{name="shop.currency(isoCode)"}
  By ISO code, e.g. `'EUR'`. Case-insensitive.
  :::

  :::field{name="shop.language(locale)"}
  By locale, e.g. `'en-GB'`.
  :::

  :::field{name="shop.salesChannel(name)"}
  By sales channel name.
  :::

  :::field{name="shop.country(iso)"}
  By ISO code, e.g. `'DE'`. Case-insensitive.
  :::

  :::field{name="shop.salutation(key)"}
  By salutation key, e.g. `'mr'`.
  :::

  :::field{name="shop.tax(rate)"}
  By numeric rate, e.g. `19`.
  :::

  :::field{name="shop.paymentMethod(technicalName)"}
  By technical name.
  :::

  :::field{name="shop.shippingMethod(technicalName)"}
  By technical name.
  :::
::

### States

::field-group
  :::field{name="shop.orderState(name)"}
  A state on the `order.state` machine, e.g. `'open'`, `'completed'`.
  :::

  :::field{name="shop.orderDeliveryState(name)"}
  A state on `order_delivery.state`, e.g. `'shipped'`.
  :::

  :::field{name="shop.orderTransactionState(name)"}
  A state on `order_transaction.state`, e.g. `'paid'`.
  :::

  :::field{name="shop.stateMachineState(machine, name)"}
  Any state, by machine and technical name.
  :::
::

### Direct context access

::field-group
  :::field{name="shop.context()" type="=&gt; ShopContext"}
  The resolved shop context. Only available while definitions are being resolved.
  :::

  :::field{name="shop.extensions" type="ShopContextExtensions"}
  Data contributed by plugins. Only available during resolution.
  :::
::

## `createShopwareClient(connection)`

Create an authenticated Admin API client from a connection (`{ url, clientId, clientSecret }`). Returns a `ShopwareClient` you can pass to the sink and context fetchers.

```ts
const client = createShopwareClient(loaded.connection)
```

Also exported: `withRetry`, a helper that retries a client call under transient failures.

## `createSyncSink(connection, options?)`

Create the sink that writes to Shopware through the Sync API. A **sink** has two methods, `write(entity, records)` and `delete(entity, ids)`, and is what the engine calls to persist and remove data.

::field-group
  :::field{name="connection" type="ShopwareConnection"}
  The shop connection.
  :::

  :::field{name="options.client" type="ShopwareClient"}
  Reuse an existing client instead of creating one.
  :::

  :::field{name="options.retry" type="RetryOptions"}
  Retry behavior for sync calls.
  :::
::

```ts
const sink = createSyncSink(loaded.connection, { client })
```

Writes go to `/_action/sync` with queue indexing, one request per entity (an upsert of the changed records). There is a per-entity request limit of **5 MB** (`ENTITY_REQUEST_BYTE_LIMIT`); exceeding it raises a `ShopwareApiError` that asks you to reduce the batch.

## `validateConnection(connection)`

Check that a connection can authenticate and reach the shop. Returns an error object if it can't, rather than throwing. It is used by `init` to validate credentials before scaffolding.

```ts
const error = await validateConnection({ url, clientId, clientSecret })
if (error) { /* handle unreachable / unauthorized shop */ }
```

## Shop context types

The indexed snapshot Fakeware fetches at the start of a run. Useful for typing advanced code that reads `shop.context()`.

::field-group
  :::field{name="ShopContext"}
  The full context: all the data arrays plus an `index` of lookup maps.
  :::

  :::field{name="ShopContextData"}
  The raw data: `currencies`, `languages`, `salesChannels`, `countries`, `salutations`, `stateMachineStates`, `taxes`, `paymentMethods`, `shippingMethods`, `extensions`.
  :::

  :::field{name="ShopContextCurrency / …Tax / …Country / …"}
  The row types for each collection.
  :::
::

Defaults are resolved sensibly: the default sales channel is the first active one; the default currency is the system default (or the sales channel's currency); the default tax is the highest rate.

## Errors

The subpath also exports the Shopware error types (`ShopwareApiError`, `ShopwareConnectionError`, `ShopContextError`) and the parsed error shape `ParsedApiError`. See the [error reference](/docs/usage/reference/errors) for what each means and how to resolve it.
