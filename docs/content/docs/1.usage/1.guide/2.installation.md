## Prerequisites

Fakeware is a Node/TypeScript CLI. You'll need:

::field-group
  :::field{name="Node.js" type="&gt;= 20"}
  Required to run the CLI.
  :::

  :::field{name="A package manager" type="npm | pnpm | yarn | bun"}
  Any of these works. Fakeware auto-detects which one you use.
  :::

  :::field{name="A Shopware 6 shop" type="URL + integration credentials"}
  Any reachable Shopware 6 store you have admin access to. See [Connect to a shop](#connect-to-a-shop).
  :::
::

## Scaffold a project

The fastest way to start is the `create-fakeware` scaffolder. It creates a `package.json`, a typed `fakeware.config.ts`, and an `.env` for your secrets, then installs the config types.

::code-group
```bash [npm]
npm create fakeware@latest
```

```bash [pnpm]
pnpm create fakeware
```

```bash [yarn]
yarn create fakeware
```

```bash [bun]
bun create fakeware
```
::

The scaffolder is interactive: it asks where to create the project, which package manager to use, and (optionally) walks you through connecting to your shop right away, validating the connection before it writes anything.

::tip
Already inside a project? Run `fakeware init` to scaffold into the current directory. It accepts the same options as the scaffolder. See the [CLI reference](/docs/usage/reference/cli#fakeware-init).
::

### Non-interactive scaffolding

For CI or repeatable setups, pass everything up front so nothing prompts:

```bash [Terminal]
fakeware init \
  --url https://my-shop.example.com \
  --client-id "$SHOPWARE_CLIENT_ID" \
  --client-secret "$SHOPWARE_CLIENT_SECRET" \
  --secrets env \
  --yes
```

## Connect to a shop

Fakeware authenticates as a Shopware **integration** using OAuth2 client credentials. To create one:

::steps{level="3"}

### Open the admin

Log in to your Shopware admin and go to **Settings → System → Integrations**.

### Create an integration

Add a new integration and give it admin privileges. Shopware generates a **client ID** and **client secret**.

### Store the credentials

Put them in your project's `.env`, alongside your shop URL:

```bash [.env]
SHOPWARE_URL=https://my-shop.example.com
SHOPWARE_CLIENT_ID=your-client-id
SHOPWARE_CLIENT_SECRET=your-client-secret
```

::

Your `fakeware.config.ts` references these by name, so secrets never live in tracked code:

```ts [fakeware.config.ts]
import { defineConfig } from '@fakeware/core/config'

export default defineConfig({
  shopware: {
    url: '$SHOPWARE_URL',
    clientId: '$SHOPWARE_CLIENT_ID',
    clientSecret: '$SHOPWARE_CLIENT_SECRET',
  },
})
```

::note
The `$NAME` syntax is resolved from your environment and `.env` at load time. See [Configuration → Environment variables](/docs/usage/reference/configuration#environment-variables) for the exact rules.
::

## A throwaway shop with Docker

No shop to test against? Boot a full Shopware 6 stack with [dockware](https://dockware.io), no compose file needed:

```bash [Terminal]
docker run --rm -d -p 80:80 --name shopware dockware/shopware:latest
```

- Storefront: `http://localhost`
- Admin: `http://localhost/admin` (default login `admin` / `shopware`)

Create an integration in that admin (as above), and point your config at `http://localhost`.

## Verify the CLI

Once installed, check the CLI is available:

```bash [Terminal]
fakeware --help
fakeware --version
```

You're ready to seed. Head to the [Quickstart](/docs/usage/guide/quickstart).
