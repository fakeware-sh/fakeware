## Prerequisites

::field-group
  :::field{name="Bun"}
  The package manager and test runner for the monorepo. See [bun.sh](https://bun.sh).
  :::

  :::field{name="Node"}
  Fakeware targets Node 20+; the published packages run on Node.
  :::

  :::field{name="Docker"}
  Optional, for a throwaway Shopware shop to develop against.
  :::
::

## Getting started

```bash [Terminal]
git clone https://github.com/fakeware-sh/fakeware.git
cd fakeware
bun install
```

## Development workflow

All commands run from the repo root and fan out across every workspace:

::field-group
  :::field{name="bun run dev"}
  Build all packages in watch mode (`tsdown --watch`).
  :::

  :::field{name="bun run build"}
  Build all packages once.
  :::

  :::field{name="bun run test"}
  Run the test suite across all packages.
  :::

  :::field{name="bun run typecheck"}
  Type-check all packages.
  :::

  :::field{name="bun run check"}
  Biome lint/format check plus typecheck. This is what CI runs.
  :::

  :::field{name="bun run fix"}
  Auto-fix with Biome, then typecheck.
  :::

  :::field{name="bun run clean"}
  Remove all build artifacts and `node_modules`.
  :::
::

::tip
Run `bun run check` (or `bun run fix`) before pushing. CI runs the same checks, so this catches failures early.
::

## Working on the CLI

Run the whole repo in watch mode so every package and app rebuilds on change, then invoke the binary:

```bash [Terminal]
bun run dev
fakeware --help
```

The published binary `fakeware` points at `apps/cli/dist/index.mjs`. On install the repo links the `fakeware` bin locally, so you can invoke it directly after a build.

## A local Shopware shop with Docker

For a throwaway shop to develop against, run the official Shopware image from [dockware](https://dockware.io), no compose file needed:

```bash [Terminal]
docker run --rm -d -p 80:80 --name shopware dockware/shopware:latest
```

- Storefront: `http://localhost`
- Admin: `http://localhost/admin` (default login `admin` / `shopware`)

Create an integration under **Settings → System → Integrations** to get a client ID and secret, then point Fakeware at `http://localhost`. See [Installation → Connect to a shop](/docs/usage/guide/installation#connect-to-a-shop) for the full flow.
