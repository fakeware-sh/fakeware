## Testing

Tests use Bun's built-in runner and live **next to the source** they cover as `*.test.ts`:

```bash [Terminal]
bun run test
```

When you add behavior, add a `*.test.ts` beside it. Coverage runs with `bun run test:coverage`.

Plugin authors have a dedicated testing kit. See [Plugin development → Testing](/docs/development/plugins/testing).

## Commits and releases

- Use [Conventional Commits](https://www.conventionalcommits.org/) (`feat:`, `fix:`, `docs:`, …). The commit type drives versioning.
- Releases are automated with [release-please](https://github.com/googleapis/release-please); you don't bump versions or edit changelogs by hand.

::note
CI runs the same `bun run check` and `bun run test` you run locally, so a green local run should mean a green pipeline.
::
