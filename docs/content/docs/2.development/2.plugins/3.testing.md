`@fakeware/core/testing` lets you exercise hooks and fetchers without a live shop or the CLI:

```ts
import { describe, expect, test } from 'bun:test'
import { createTestPluginContext, runPluginHookOnce } from '@fakeware/core/testing'
import { myPlugin } from '../src/plugin'

test('configResolved accepts a valid config', async () => {
  const plugin = myPlugin()
  const ctx = createTestPluginContext({ name: plugin.name })
  await runPluginHookOnce(plugin, 'configResolved', ctx)
})
```

::field-group
  :::field{name="createTestPluginContext(opts?)"}
  Builds a `PluginContext` with sensible defaults. Override any field (`config`, `connection`, `shopContext`, `shopData`, `sink`, …) via `opts`.
  :::

  :::field{name="runPluginHookOnce(plugin, hook, ctx)"}
  Invokes one hook (`configResolved`, `contextReady`, `beforeApply`, or `beforeRevert`) with the context; a no-op if the plugin doesn't define it.
  :::

  :::field{name="createCollectingLogSink(debug?)"}
  A log sink that records everything the plugin logs, so you can assert on `info` / `warn` / `error` output.
  :::
::

To test a fetcher directly, call its `fetch` with a stub client and its `merge` against a context data object, then assert on `data.extensions`:

```ts
test('warehouse fetcher merges rows into extensions', async () => {
  const raw = await warehousesFetcher.fetch(stubClient)
  const data = { extensions: {} } as ShopContextData
  warehousesFetcher.merge(data, raw)
  expect(data.extensions.pickwareWarehouses).toHaveLength(2)
})
```
