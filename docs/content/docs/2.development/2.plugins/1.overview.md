A plugin extends Fakeware without forking it. Plugins can **fetch extra data** from the shop (so your definitions can reference it) and **hook into the run lifecycle** (to validate, log, or run side effects around `up` and `down`). The [`@fakeware/plugin-pickware`](https://github.com/fakeware-sh/fakeware/tree/main/plugins/plugin-pickware) package is the reference implementation.

## `definePlugin`

A plugin is a plain object. `definePlugin` is an identity helper that types it:

```ts
import { definePlugin, type FakewarePlugin } from '@fakeware/core'

export function myPlugin(): FakewarePlugin {
  return definePlugin({
    name: 'my-plugin',
    fetchers: [/* ShopContextFetcher[] */],
    hooks: {/* PluginHooks */},
  })
}
```

::field-group
  :::field{name="name" type="string"}
  Required and **unique** across the registered plugins. Used in error messages and logs.
  :::

  :::field{name="fetchers" type="ShopContextFetcher[]"}
  Optional. Extra data to fetch from the shop and merge into the shop context.
  :::

  :::field{name="hooks" type="PluginHooks"}
  Optional. Functions called at specific points in a run.
  :::
::

Exposing the plugin as a factory (`myPlugin()`) is the convention, so users register it with `myPlugin()` and you have a place to accept options later.

## Registering a plugin

Users add plugins to the `plugins` array in `fakeware.config.ts`:

```ts [fakeware.config.ts]
import { defineConfig } from '@fakeware/core/config'
import { pickware } from '@fakeware/plugin-pickware'

export default defineConfig({
  shopware: { /* … */ },
  plugins: [pickware()],
})
```

At load time Fakeware validates each entry: `name` must be a non-empty string, `fetchers` (if present) an array, and every key under `hooks` a known hook name. **Duplicate names throw**, and a legacy `setup` field is rejected in favor of `hooks.contextReady`.

::tip
`fakeware init` offers the official plugins in an interactive step and wires the selected ones for you: the import, the `plugins` entry, and the `package.json` dependency. Pass `--plugins <ids>` (or `--no-plugins`) to pick them non-interactively.
::
