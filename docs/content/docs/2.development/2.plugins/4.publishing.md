Publish a plugin as its own package that depends on `@fakeware/core` as a **peer dependency**, and add the `fakeware-plugin` keyword to `package.json` so it's discoverable:

```json [package.json]
{
  "name": "@you/fakeware-plugin-thing",
  "keywords": ["fakeware-plugin"],
  "peerDependencies": {
    "@fakeware/core": ">=0.0.9"
  }
}
```

::note
Using a peer dependency (rather than a regular dependency) ensures the plugin runs against the **same** `@fakeware/core` the host project uses, so context types and tokens line up.
::

Point users at your plugin's factory in their config:

```ts [fakeware.config.ts]
import { thing } from '@you/fakeware-plugin-thing'

export default defineConfig({
  shopware: { /* … */ },
  plugins: [thing()],
})
```
